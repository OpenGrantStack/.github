# Contributing to GrantReady Documentation

Thank you for your interest in contributing to the GrantReady Documentation repository. This document provides guidelines and processes for contributing templates, schemas, and documentation.

## Purpose of Contributions

We welcome contributions that:

1. Enhance compliance with government regulations
2. Improve accessibility for diverse user groups
3. Add templates for new grant types or jurisdictions
4. Clarify audit requirements and processes
5. Improve schema interoperability

## Contribution Types

### Documentation
- Compliance guidelines
- Audit preparation materials
- Best practices for grant management
- Regulatory updates

### Templates
- Grant application forms
- Reporting templates
- Compliance certification documents
- Audit worksheets

### Schemas
- JSON schemas for grant data
- Validation rules
- Data interchange formats

## Contribution Process

### 1. Preliminary Review
Before starting significant work, please:
- Review existing issues to avoid duplication
- Consider opening a discussion issue for feedback
- Ensure your contribution aligns with repository goals

### 2. Creating Contributions

#### For Documentation:
- Use clear, professional language
- Cite regulatory sources where applicable
- Structure content with appropriate headings
- Include practical examples when helpful

#### For Templates:
- Maintain consistent formatting
- Include all necessary legal disclaimers
- Ensure mobile-friendly design
- Provide clear instructions for use

#### For Schemas:
- Follow JSON Schema specification
- Include comprehensive validation
- Document all fields and requirements
- Test with sample data

### 3. Submission Requirements

All submissions must include:

1. **Complete files** with all necessary content
2. **Descriptive commit messages** explaining changes
3. **Reference to regulations** where applicable
4. **Testing information** for schemas
5. **Version information** for templates

### 4. Review Process

1. **Initial Review**: Within 7 business days
2. **Feedback Cycle**: Iterative improvements as needed
3. **Final Approval**: By repository maintainers
4. **Merge & Documentation**: Integration and attribution

## Technical Requirements

### File Structure
- Place documentation in `/docs/`
- Place templates in `/templates/`
- Place schemas in `/schemas/`
- Use kebab-case for filenames (e.g., `grant-application-template.md`)

### Formatting Standards

#### Markdown Documentation:
- Use ATX-style headers (#, ##, ###)
- Wrap lines at 80 characters for prose
- Use tables for comparative information
- Include back to top links for long documents

#### JSON Schemas:
- Include `$schema` declaration
- Provide comprehensive `description` fields
- Use consistent property ordering
- Include examples for complex structures

### Versioning
- Templates should include version numbers in metadata
- Schemas should follow semantic versioning
- Breaking changes require major version increments

## Legal Considerations

### Regulatory Compliance
- Verify alignment with current regulations
- Note jurisdiction-specific requirements
- Include necessary legal disclaimers
- Respect data privacy regulations

### Licensing
- All contributions are licensed under Apache 2.0
- Contributors retain copyright but grant license
- Ensure no proprietary information is included
- Attribute third-party content appropriately

## Quality Standards

### Accuracy
- Verify all regulatory references
- Test template functionality
- Validate schema compliance
- Review for technical correctness

### Accessibility
- Ensure mobile-first design
- Support screen readers
- Provide alternative text
- Maintain color contrast standards

### Usability
- Include clear instructions
- Provide examples
- Consider diverse user needs
- Support multiple languages where possible

## Getting Help

- Open an issue for questions
- Reference existing examples
- Consult documentation guidelines
- Contact maintainers for complex queries

## Recognition

Contributors will be:
- Listed in release notes
- Acknowledged in documentation
- Included in contributor files
- Recognized for significant contributions
